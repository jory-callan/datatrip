# AGENTS.md — 项目入口 + AI 工作规则

> 新 AI 进入本项目的第一个文件。读完本文件就知道这是什么项目、怎么启动、该怎么工作。

---

## 项目简介

**Jerry DB Manager** — 企业统一数据管理平台。
产品定位：  Data Ops Platform（数据运维平台） — 不是又一个数据库管理工具，而是统一的数据操作网关 +
  审批治理层。核心价值是"所有数据操作经过审批、有审计、可追溯"，不是替代数据库自身的权限体系。
包括 标准sql， nosql, MQ等各种数据源对接。

## AI 工作规则

### Git 提交

按 `type(scope): subject` 格式，注释尽量详细。

```
feat(web): 添加表数据预览功能
fix(server): 修复连接池泄漏问题
refactor(auth): 提取权限判断到统一 switch
```

### 代理与网络

- GitHub clone 加 `https://gh-proxy.com/` 前缀
- 搜索用 bing.com

### 沟通风格

- 给出专业判断，不迎合
- 用户需求有问题时指出并提供更好方案
- 永远先确认完整计划，等待 "start" 再写代码

### 知识沉淀（关键规则）

用户在对话中表达以下内容时，AI **必须**立即更新 `CONVENTIONS.md`，**不需要**等待用户指示：

| 触发场景 | 更新位置 |
|---------|---------|
| 技术选型偏好（"用 X 不用 Y"） | 对应章节约定表格 |
| 弃用/重构决策（"以后别再用 Z"） | 对应章节约定表格 |
| 架构决策（"双库隔离"） | 决策记录 |
| 编码习惯（"密码字段 json:'-' "） | 对应章节约定表格 |
| 项目边界变化（"支持 Oracle"） | 对应章节 + 决策记录 |

**原则**：多记比漏记好。每条带日期。

### 自动优化工作流程

AI 在开发过程中如果发现以下问题，**应主动提出优化方案**，无需等待用户要求：

| 发现场景 | 示例 |
|---------|------|
| 重复性手动操作 | "你每次都手动改配置，可以写个 init.sh" |
| 过时的脚本/工具 | "dev.sh 还在用 PID 文件，端口更可靠" |
| 低效的开发流程 | "每次改代码都 build 再测试，可以用 air 热重载" |
| 缺失的自动化 | "数据库迁移要手写 SQL，可以集成自动迁移" |
| 不一致的写法 | "有的 handler 用 global.DB，有的自己 new，该统一" |

**工作方式**：发现 → 指出问题 → 给出方案 → 问用户要不要做。用户同意后再执行。

### 一致性检查

AI 在修改代码时，如果发现**代码实际写法与 `CONVENTIONS.md` 的约定不符**，应主动指出不一致并询问是否需要更新文档：

| 发现场景 | 示例 |
|---------|------|
| 约定过时 | "CONVENTIONS.md 说用 ofetch，但代码里用的是 axios" |
| 约定未落地 | "CONVENTIONS.md 说密码字段 json:'-'，但 user.go 里没加" |
| 代码中有新模式 | "我看到新模块用了 wire DI，但 CONVENTIONS 说不用 DI 容器" |
| 约定遗漏 | "代码里所有 handler 都用了统一错误码，但 CONVENTIONS 没记录这个约定" |

**原则**：宁愿多报一条误报，也不要默默放跑一个不一致。

### 决策记录维护

决策记录会随时间增长，AI 在添加新决策时顺带维护：

- 超过 6 个月的初始化性记录（如"项目初始化"），可以归档到 `docs/archive/`
- 只保留**近期的有效决策**，保持文件精炼
- 归档时机：当你追加新记录时，顺手把最旧的几条移走

```
CLAUDE.md / AGENTS.md ← 你在这里（5 分钟）→ 项目介绍 + AI 工作规则
    ↓
CONVENTIONS.md（5 分钟） → 系统架构 + 决策记录
    ↓ (按需)
server/CONVENTIONS.md  → 改后端前读
web/CONVENTIONS.md     → 改前端前读
docs/                  → 深度参考（permission / sql-execution / database-extension）
```

### 必做动作
docs/api 是api约定文件夹，里面包含了前后端协作的api约定以及接口逻辑。用户需要排查错误优先查看此文件夹，新功能完成需要及时维护此文件夹
